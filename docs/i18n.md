# Internationalization (i18n) Guide

This project uses `vue-i18n` (v9) for handling internationalization.

## Architecture

- **Library**: `vue-i18n` (Composition API mode)
- **Locale Files**: Located in `src/locales/{lang}.ts`.
- **Configuration**: `src/i18n/index.ts`.
- **State Management**: `src/composables/useLanguage.ts` handles language switching and persistence.

## Adding a New Language

1. Create a new file in `src/locales/` (e.g., `ja-JP.ts`).
2. Copy the structure from `en-US.ts` and translate the values.
3. Import the new locale in `src/i18n/index.ts` and add it to the `messages` object.
4. Update `src/composables/useLanguage.ts` to include the new language in `languageOptions`.

## Usage in Components

### Script Setup

```typescript
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()

const title = computed(() => t('home.title'))
</script>
```

### Template

```html
<template>
  <div>{{ t('home.welcome', { name: 'User' }) }}</div>
</template>
```

## Features

### Interpolation
```typescript
// en-US.ts
welcome: 'Hello, {name}!'

// Component
t('welcome', { name: 'Alice' }) // -> Hello, Alice!
```

### Pluralization
```typescript
// en-US.ts
car: 'car | cars'

// Component
t('car', 1) // -> car
t('car', 2) // -> cars
```

### Date Formatting
We have configured global datetime formats.

```typescript
d(new Date(), 'short') // -> Oct 24, 2023 (en-US)
```

### Number Formatting
```typescript
n(1000, 'currency') // -> $1,000.00 (en-US)
```

## Best Practices

1. **Keys**: Use nested keys for better organization (e.g., `page.section.key`).
2. **Variables**: Use named placeholders `{name}` instead of list-based `{0}` for clarity.
3. **Extraction**: Use VS Code extension "i18n Ally" to manage and extract translations efficiently.

## Testing

Ensure to mock `useI18n` in your unit tests if you are testing components that rely on it.

```typescript
// Example vitest mock
vi.mock('vue-i18n', () => ({
  useI18n: () => ({
    t: (key: string) => key
  })
}))
```
